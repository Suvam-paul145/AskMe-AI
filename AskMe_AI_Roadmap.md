# 1. Executive Summary

## Project Overview

**AskMe AI** is an intelligent, AI-powered personal learning companion that transforms how students engage with their own study materials. By combining the power of large language models, retrieval-augmented generation, and adaptive learning science, AskMe AI turns static notes and PDFs into dynamic, interactive learning experiences — tailored uniquely to every student.

## Vision

> *"A world where every student — regardless of geography, income, or language — has access to a world-class personal tutor available 24/7."*

## Mission

To democratize personalized education by building an AI-powered study system that understands each student's unique learning needs, adapts to their weaknesses, and accelerates their path to mastery — affordably and accessibly.

## Main Objective

- Enable students to upload their own study material and instantly get AI-generated summaries, quizzes, doubt resolution, and personalized study plans.

# 2. Problem Statement

## Problems Faced by Students

### 1. Lack of Personalized Learning
Every student learns differently — some need visual aids, others prefer text summaries, still others learn through repeated practice. Yet every classroom, textbook, and online course delivers a one-size-fits-all experience. There is no system that studies *how a student studies* and adapts accordingly.

### 2. Expensive Private Tutors
A good private tutor in India costs ₹500–₹3,000 per hour. In the US, this ranges from $30–$150/hour. For the majority of students globally, this is financially inaccessible. Even students who can afford tutors face scheduling conflicts, inconsistent quality, and limited subject coverage.

### 3. Information Overload
Students today are bombarded with YouTube videos, PDFs, notes, slides, textbooks, and online resources. Having too much information without structure leads to paralysis, confusion, and poor retention. Students don't need more content — they need intelligent content curation and synthesis.

### 4. Weak Revision Systems
Most students revise passively — re-reading notes, which has one of the lowest knowledge retention rates (~10–20%). Active recall through quizzes and spaced repetition dramatically improves retention, yet very few students have access to tools that automate this intelligently.

### 5. Difficulty Understanding Notes
Students often take poor-quality notes in class or struggle to understand dense textbook language. Without someone to explain concepts clearly, they either skip difficult topics or develop misconceptions that compound over time.

### 6. Poor Doubt-Solving Accessibility
When a student has a doubt at 11 PM the night before an exam, there is no teacher available. Online forums (Reddit, Quora) give generic answers. ChatGPT gives answers but not in the context of the student's own notes or syllabus. The gap between having a doubt and getting a relevant, contextual answer is enormous.

### 7. No Intelligent Progress Tracking
Students study without a clear map of where they are, what they've mastered, and what still needs work. Traditional exams provide feedback too late and too infrequently.

# 6. Core Features

## Mandatory Features

### 1. AI Notes Summarizer
- Accepts uploaded PDFs, DOCX, images, or typed notes
- Generates tiered summaries: 1-paragraph overview, key-points list, detailed breakdown
- Highlights the most exam-relevant concepts
- Supports "TL;DR" mode and "Deep Dive" mode
- Creates chapter-wise and topic-wise summaries

### 2. AI Doubt Solver Chatbot
- Context-aware chatbot grounded in the student's uploaded documents (RAG)
- Answers questions directly from the student's notes, not generic internet content
- Supports follow-up questions in the same session
- Provides step-by-step explanations
- Tracks frequently asked questions per topic for analytics

### 3. Quiz Generator
- Auto-generates MCQs, True/False, Fill-in-the-Blank, Short Answer questions
- Difficulty levels: Easy, Medium, Hard, Exam-level
- Topic-specific or full-chapter quiz modes
- Randomized question order to prevent memorization of sequences
- Instant answer feedback with explanations

### 4. Auto Evaluation System
- Evaluates student answers with AI-driven grading for subjective questions
- Provides percentage score + topic-wise breakdown
- Flags incorrect answers and suggests the correct concept
- Stores all quiz results for longitudinal performance analysis

### 5. Weak Topic Detection
- Analyzes quiz performance patterns across sessions
- Identifies topics where accuracy consistently falls below threshold
- Generates a "Weak Topics Report" per student
- Automatically increases quiz frequency for weak areas (spaced repetition)
- Sends targeted revision reminders

### 6. Personalized Study Plan Generator
- Takes exam date, available study hours/day, and weak topics as input
- Creates a day-wise, topic-wise revision schedule
- Adapts dynamically as performance improves
- Integrates with calendar for study session reminders
- Highlights priority topics based on exam proximity

### 7. Dashboard Analytics
- Visual graphs: quiz scores over time, topics mastered vs. pending, time spent
- Weekly and monthly performance summaries
- Streak counter and XP points display
- Subject-wise performance breakdown
- Recommended next action (study/quiz/review)

### 8. Learning Progress Tracking
- Tracks every interaction: quiz attempts, chat queries, notes read
- Generates a learning velocity score
- Shows estimated time to mastery for each topic
- Historical comparison: "You scored 40% last week, 75% this week"

### 9. Multi-language Support
- Summaries and explanations available in Hindi, Tamil, Bengali, Telugu, Marathi, and other Indian languages
- Quiz questions available in selected language
- UI available in multiple regional languages
- Translation powered by DeepL or Google Translate API

### 10. Simple Explanation Mode
- Rewrites complex academic text in plain, everyday language
- Uses analogies and examples to make abstract concepts concrete
- Ideal for students encountering a topic for the first time
- Adjustable reading level: Grade 6 / Grade 10 / Undergraduate

### 11. Exam Mode
- Timed mock exam environment simulating real exam conditions
- Questions drawn from the student's own uploaded material
- No hints or AI help during exam mode
- Full performance report generated at the end
- Benchmarks against previous attempts

### 12. Deep Explanation Mode
- Provides exhaustive, textbook-level explanations for selected topics
- Includes related concepts, prerequisites, and advanced extensions
- Cites relevant passages from the student's uploaded documents
- Ideal for conceptual mastery before competitive exams

## Detailed Workflow: Upload Notes

```
Student clicks "Upload Notes"
        │
        ▼
Select file (PDF / Image / DOCX / Text)
        │
        ▼
Client-side validation (size < 10MB, format check)
        │
        ▼
File uploaded to secure cloud storage (S3)
        │
        ▼
Backend triggers preprocessing job:
    ├─ If PDF → extract text via pdfplumber
    ├─ If Image → OCR via Tesseract / Google Vision
    └─ If DOCX → extract via python-docx
        │
        ▼
Text cleaned (remove headers/footers, fix encoding)
        │
        ▼
Text chunked into 500-token segments with 50-token overlap
        │
        ▼
Each chunk sent to OpenAI Embedding API → vector generated
        │
        ▼
Vectors stored in Pinecone / ChromaDB with metadata
    (user_id, document_id, chunk_id, page_number)
        │
        ▼
Document record created in PostgreSQL
        │
        ▼
Student notified: "Your notes are ready! Tools generated."
        │
        ▼
AI Summary, Flashcards, and Quiz auto-generated in background
```

## Detailed Workflow: Quiz Generation

```
Student clicks "Generate Quiz" on a document
        │
        ▼
Student selects: Topic, Difficulty, Question Count, Type
        │
        ▼
Backend queries vector DB → retrieves top-k relevant chunks
        │
        ▼
Chunks assembled into context window
        │
        ▼
Prompt sent to LLM:
    "Based on the following content, generate 10 MCQ questions
     at Medium difficulty. Format as JSON with:
     question, options[A,B,C,D], correct_answer, explanation"
        │
        ▼
LLM returns JSON quiz data
        │
        ▼
Quiz validated (format, uniqueness, answer accuracy check)
        │
        ▼
Quiz stored in database (linked to document + user)
        │
        ▼
Quiz rendered in interactive UI
        │
        ▼
Student takes quiz → answers submitted
        │
        ▼
Auto Evaluation: correct answers compared, score calculated
        │
        ▼
Per-question feedback generated by AI
        │
        ▼
Results stored → Weak Topic Detection pipeline triggered
        │
        ▼
Dashboard analytics updated
```

## Detailed Workflow: Weak Topic Detection

```
After every quiz attempt:
        │
        ▼
Quiz result stored: {user_id, topic, question_id, correct: bool}
        │
        ▼
Aggregate analysis: last 5 quiz attempts per topic
        │
        ▼
Accuracy calculated per topic:
    If accuracy < 60% for 2+ consecutive attempts → WEAK
        │
        ▼
Weak topic record created/updated in database
        │
        ▼
Study plan recalculated: weak topics scheduled for next 3 days
        │
        ▼
AI Recommendation generated: "Focus on [Topic] today"
        │
        ▼
Push notification sent: "Your weak topic alert: [Topic]"
        │
        ▼
Weak Topics section in Dashboard updated with visual indicators
```

## Retrieval-Augmented Generation (RAG) Pipeline

```
Document Upload
     │
     ▼
Text Extraction (PDF/OCR/DOCX)
     │
     ▼
Text Cleaning & Normalization
     │
     ▼
Chunking Strategy:
  - Chunk size: 500 tokens
  - Overlap: 50 tokens
  - Boundary: sentence-aware (no mid-sentence cuts)
     │
     ▼
Embedding Generation:
  - Model: text-embedding-3-small
  - Dimension: 1536
  - Batch processing: 100 chunks/request
     │
     ▼
Vector Storage (Pinecone / ChromaDB):
  - Index: user-specific namespace
  - Metadata: {doc_id, chunk_id, page, topic_tag}
     │
     ▼
At Query Time:
  - Query embedded → similarity search (cosine)
  - Top-5 chunks retrieved
  - Relevance threshold: 0.75
  - Assembled into context window
     │
     ▼
LLM Generation with context-grounded prompt
```

## Quiz Generation Pipeline

```
1. Select topic from document
2. Retrieve relevant chunks (top-10 for more variety)
3. Build structured prompt with difficulty + type params
4. LLM generates JSON quiz
5. JSON validated with Pydantic schema
6. Duplicate question check (semantic similarity)
7. Answer correctness verification (second LLM call)
8. Quiz stored in DB + rendered to student
```

## Weak Topic Analysis Pipeline

```
Per Quiz Attempt:
  - Store: {user_id, topic, question_id, is_correct, timestamp}

Aggregation (runs daily or after each quiz):
  - Group by user_id + topic
  - Calculate rolling accuracy (last 5 attempts)
  - Apply threshold: accuracy < 60% = WEAK

Prediction Enhancement:
  - Track time-to-answer (slow = uncertain)
  - Track "hint used" events
  - Weight recent attempts more heavily (decay factor: 0.8)

Output:
  - Weak topics list with confidence score
  - Recommended study duration per topic
  - Adaptive quiz weight adjustment
```

## Hallucination Reduction Strategy

1. **RAG Grounding** — AI only uses content from uploaded documents
2. **Temperature Control** — Set temperature to 0.3 for factual tasks (quiz, summary)
3. **Citation Injection** — AI cites specific page/section for each answer
4. **Confidence Thresholding** — If retrieval score < 0.65, AI warns "Low confidence"
5. **Answer Verification** — Second LLM pass verifies quiz correct answers
6. **System Prompt Constraints** — Explicit instruction to not invent information
7. **Human Feedback Loop** — Students can flag incorrect AI answers

# 20. Scalability Plan

## Infrastructure Scaling

### Application Layer
- **Horizontal scaling** via container orchestration (Kubernetes)
- **Auto-scaling** based on CPU/memory metrics (AWS ECS auto-scaling)
- **Load balancing** via AWS ALB / Nginx
- **CDN** for static assets (CloudFront / Cloudflare)

### AI Layer
- **Caching** — Cache embeddings per document (Redis); avoid re-embedding
- **Batch processing** — Process 100 chunks per API call (not 100 separate calls)
- **Model tiering** — GPT-3.5 for simple tasks, GPT-4o for complex reasoning
- **Rate limiting** — Per-user and per-tier quota management
- **Async processing** — AI tasks run in background (Celery workers)

### Database Layer
- **Read replicas** for analytics queries
- **Connection pooling** (PgBouncer)
- **Partitioned tables** for quiz_answers and user_analytics (date-based)
- **Redis cache** for session data and frequently-accessed summaries

### Vector DB Layer
- **Pinecone** with namespace-per-user isolation
- **Shard** by region for latency reduction
- **Metadata filtering** to reduce retrieval scope

## Global Deployment
- Multi-region: India (Mumbai), US (Virginia), Europe (Frankfurt)
- Data residency compliance: Indian data stays in Indian servers
- Edge caching via Cloudflare for sub-100ms landing page loads

## User Scale Projections

| Users | Infrastructure Requirement |
|---|---|
| 0–1K | Single server + Supabase |
| 1K–10K | 2-3 servers + managed DB |
| 10K–100K | Kubernetes cluster + Redis + Pinecone |
| 100K–1M | Multi-region + CDN + read replicas |
| 1M+ | Full microservices + global distribution |

# 21. Security & Privacy

- JWT tokens with 15-minute expiry + 7-day refresh tokens

- MFA support (TOTP) for Pro+ users

|---|---|---|---|
| AI hallucination in quiz answers | High | High | RAG grounding + answer verification |
| OpenAI API cost overrun | Medium | High | Usage quotas per tier + caching |
| PDF parsing failure | Medium | Medium | Multiple fallback parsers |

| Competitor copies idea | High | Medium | Rapid feature development + brand moat |
| API rate limits | Medium | Medium | Multiple provider fallbacks |
| Poor OCR accuracy | Medium | Medium | Multiple OCR engines |

## Detailed Mitigation Strategies

### Hallucination Risk
- RAG ensures answers come from student's own document
- Temperature setting 0.3 for factual tasks
- "Confidence score" shown with each AI answer
- "Report incorrect answer" button for user feedback
- Second-pass verification for quiz correct answers

### API Cost Management
- Aggressive caching (embeddings never re-computed for same document)
- Usage monitoring dashboard with alerts at 80% budget
- Model tiering: cheap models for simple tasks
- Batch API calls to reduce per-token cost
- Monthly cost cap enforcement per user tier

### Scaling Risks
- Architecture designed for horizontal scaling from Day 1
- Load testing before public launches
- Gradual rollouts (10% → 30% → 100% traffic)
- Feature flags for quick rollback
